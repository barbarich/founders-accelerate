## Слайд 5
**События на пикселе · что и куда**
События на пикселе · что вешаем и куда

Пять событий покрывают 95% продуктов в когорте

Standard Events Meta · выбираешь под свою воронку

Purchase

На thank-you page · после оплаты

Главное событие для e-commerce / SaaS. Параметры: value · currency · content_ids. Без value Meta не может оптимизировать на ROAS.

для · Mila (если есть платный план)

Lead

После отправки формы / вейтлист submit

Главное событие для lead-gen. Используется когда продажа далеко (B2B, дорогая подписка, демо-сейлз).

для · Vlad · Lea · Laura

CompleteRegistration

После создания аккаунта / signup

Для freemium / триал-флоу. Отличается от Lead тем, что регистрация = бóльшая глубина намерения, чем email-сбор.

для · Mila · Lea

Contact

Клик на «Написать в WhatsApp / Telegram»

Когда основной CTA = диалог в мессенджере. Вешается на onClick кнопки. Custom event если нужны детали (район/тип/бюджет).

для · Dira.click

ViewContent

Открытие ключевой страницы (продукт / прайс)

Fallback, когда покупок/лидов пока мало (< 50 в неделю). Кормишь алгоритм объёмом верхних событий, пока низ воронки не наберёт массу.

для · Все на старте

Три способа поставить событие

01Pixel script · в <head>

Base code (PageView fires auto) + ручной fbq('track', 'Purchase', {value, currency}) на success-странице. Минимум. Работает везде.

02Google Tag Manager

Без кода. Trigger «page url contains /thank-you» → Meta Pixel tag → Purchase. Удобно для Lovable / Webflow / Wix лендингов.

03Conversions API (CAPI)

Server-side. Stripe webhook / Supabase Edge Function / Shopify pixel → fetch на Meta. Обязательно для 2026 — см. следующий слайд.

## Слайд 6
**Топ-практики событий · 2026**
Топ-правила отслеживания · 18 мая 2026

Без этих шести шагов ты видишь половину правды

01Отправляй данные с сервера, не только из браузера

ЕСТЬ ДУБЛЁР · ЕСТЬ ДАННЫЕ

Блокировщики рекламы и новые правила iOS скрывают до половины покупок. Если отправлять данные напрямую с твоего сервера — их не заблокировать. Без этого ты видишь только половину правды.

02Один номер — одна покупка

ОДНО СОБЫТИЕ · НЕ ДВА

Браузер и сервер оба видят одну покупку. Дай каждой покупке свой номер — тогда система поймёт, что это одно и то же, и не посчитает дважды.

03Всегда указывай сумму покупки

БЕЗ НИХ НЕТ ПРИБЫЛИ

Если передаёшь только факт покупки, система ищет любую покупку. Если передаёшь сумму — ищет самые крупные. Для заявок указывай примерную ценность (например, $50 за качественного клиента).

04Расставь приоритеты событий

8 ПРИОРИТЕТОВ НА ДОМЕН

iOS ограничивает, сколько действий можно отследить. Зайди в настройки Events Manager и расставь приоритеты. Покупка — на первом месте. Просмотр страницы — последним или убери совсем.

05Проверь всё до запуска

5 МИНУТ · СПАСАЕТ НЕДЕЛИ

В Events Manager есть режим тестирования: проходишь сценарий покупки и видишь, приходят ли данные. Если нет — чинишь ДО запуска рекламы, а не после слива денег.

06Сверяй данные из двух источников

ДВА ИСТОЧНИКА ПРАВДЫ

Рекламный кабинет показывает клики. Google Analytics — что человек делал на сайте после. Добавь метки в ссылки рекламы, чтобы связывать оба источника.

Готовые решения · без программиста

Stripe

Оплата через Stripe → автоматически отправляет данные о покупке

Supabase Edge Function

Свой сервер для отправки данных о покупках

Stape · CAPI Gateway

Готовый сервис-прокси, $20/мес, программист не нужен

Shopify · Meta channel

Встроенная интеграция для магазинов на Shopify

## Слайд 8
**Старый vs новый Meta**
Meta-реклама · что сломалось к 2026

Старый интерфейс остался. Старая логика — нет

Не работает · 2022 way

* → Ручной таргет: возраст, пол, интересы, геолокация
* → Lookalike-аудитории на 1–10% сходства
* → A/B-тесты по два креатива в неделю
* → Месяцы оптимизации одной кампании
* → Маркетолог нужен — без него не разберёшься

Работает · 2026 way

* → Advantage+ Audience: широкий вход, Andromeda сама находит покупателя
* → Advantage+ Placements: все площадки автоматически — Feed, Reels, Stories, Audience Network
* → Dynamic Creative: 5–10 видео + 3–5 текстов, Meta перемешивает
* → Learning phase 7 дней / 50 конверсий — не трогаешь
* → Одиночка-фаундер запускает сам за вечер

Главный сдвиг: ты не настраиваешь — ты кормишь. Алгоритм решает кому показать. Ты решаешь сколько и какие креативы дать.

## Слайд 14
**Шаг 3 · Ad · Dynamic Creative**
🔴 Воркшоп · Шаг 3 · Ad · Dynamic Creative

5–10 креативов в одну группу. Meta микширует.

Грузишь медиа из домашки M9. Тексты — Claude / GPT генерит за 5 минут. Главное — объём вариантов, не идеальность одного.

Identity

Facebook Page + Instagram аккаунт. Если IG пустой — лучше выключить, чем показывать пустой профиль.

Format

Single image · Carousel · Video. Рекомендую микс: половина видео, половина статика. Andromeda сама выберет работающее.

Media

Грузим 5–10 креативов сразу. Dynamic Creative ON — Meta перемешивает заголовки + медиа + CTA.

Primary text · 3–5 вариантов

Короткий hook · длинный сториз · вопрос. Один Pain + один Outcome в каждом.

Headlines · 3–5 вариантов

Выгода / результат / соц.доказательство. Не «Mikey — приложение для знакомств». А «500 человек встретили партнёра за месяц».

CTA

Sign Up / Shop Now / Learn More / Send Message. Одно действие, без «или». Send Message — для Dira (click-to-WhatsApp).

Tracking · UTM в URL

utm_source=meta · utm_campaign=… · utm_content={{ad.name}}. Без UTM в GA4 / Mixpanel ничего не увидишь.

Preview во всех placement

Feed · Story · Reels · Right Column. Проверяем что текст не обрезается, кнопка видна. 30 секунд экономии месяца боли.

## Слайд 16
**Шаг 5 · After launch · метрики**
🔴 Воркшоп · Шаг 5 · After launch · что смотреть и когда что менять

Reports → Breakdown → правило трёх дней

Метрики, которые добавляешь в колонки

CPM

Стоимость 1000 показов. Высокий — плохой креатив или узкая аудитория. Норма B2C: $5–25.

CTR

Кликабельность. < 1% — креатив не работает, меняй. > 2% — на верном пути.

CPA

Цена за конверсию. Главная метрика. Сравниваешь с LTV. Если CPA > LTV — кампания убыточная.

Quality / Engagement / Conversion ranking

Three rankings в колонке. Below Average — выключаешь креатив. Above — масштабируешь бюджет +20%.

Playbook оптимизации

ДНИ 1–3

Не трогаешь

Любая правка сбрасывает learning. Сидишь, наблюдаешь, фиксируешь baseline.

ДНИ 4–7

Готовишься

Делаешь ещё 5 креативов для следующей итерации. Из winners понимаешь, какой угол сработал.

ПОСЛЕ 7 ДНЕЙ

Резать и доливать

Креативы с Below Average — выключаешь. На winning ad set доливаешь бюджет на +20% в день, не больше.

FREQUENCY > 3

Обновлять

Если за неделю одно лицо увидело креатив больше 3 раз — пора заливать новую партию ассетов. Иначе CPM растёт.

## Слайд 20
**Семь ошибок одиночек**
Семь ошибок, которыми ломаются первые кампании одиночек

Знаешь заранее — не повторишь. Это типовые грабли, на которые наступает каждый, кто настраивает Meta впервые.

01Запустили без работающего пикселя

Алгоритму нечего учить. Бюджет уходит, оптимизации нет. Проверка — Test Events до Publish.
