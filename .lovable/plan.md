Я проверил опубликованный сайт как crawler и нашёл две реальные причины, почему Gemini всё ещё может отвечать «страница недоступна для инструментов сканирования», несмотря на robots.txt и llms.txt.

Наблюдения:
- Публикация публичная: `effective_publish_visibility = public`.
- `robots.txt`, `llms.txt` и `sitemap.xml` доступны по прямым ссылкам.
- Главная и маршруты `/ru`, `/program`, `/register`, `/ru/mentor` возвращают `200`.
- Но статические `.md` страницы для LLM отдаются с `Content-Type: application/octet-stream`, а не как текст/markdown. Для части LLM-инструментов это выглядит как бинарный файл, и они могут отказаться парсить его.
- В коде есть страница `src/pages/Accelerator.tsx`, которая динамически добавляет `<meta name="robots" content="noindex, nofollow">`. Это противоречит задаче «все страницы доступны для просмотра и сканирования» и может мешать индексаторам, которые исполняют JavaScript.

План исправления:

1. Убрать запрет индексации со старой страницы акселератора
   - Удалить динамическое добавление `noindex, nofollow` из `src/pages/Accelerator.tsx`.
   - Оставить страницу доступной для просмотра, но без запрета для ботов.

2. Заменить `.md` LLM-файлы на URL с корректным текстовым типом
   - Добавить рядом с текущими Markdown-файлами текстовые версии с расширением `.txt`, например:
     - `/llms/mini-course-landing.txt`
     - `/llms/accelerator-landing.txt`
     - `/llms/mentor.txt`
     - `/llms/program.txt`
     - `/llms/about.txt`
     - `/mini-course/lesson1.txt`
   - На Lovable-хостинге `.txt` отдаётся как `text/plain`, что надёжнее для Gemini/LLM-сканеров, чем `.md`, который сейчас отдаётся как `application/octet-stream`.

3. Обновить AI-индекс и sitemap
   - В `public/llms.txt` заменить основные ссылки с `.md` на `.txt`.
   - Оставить `.md` как дополнительные/legacy-ссылки, чтобы не ломать уже опубликованные ссылки.
   - В `public/sitemap.xml` добавить `.txt` версии и ключевые публичные маршруты.
   - В `public/robots.txt` оставить `Allow: /`, но ссылку на `llms.txt` сохранить только как вспомогательную; основной sitemap оставить XML.

4. Усилить статический HTML-фолбэк в `index.html`
   - Добавить явный блок для сканеров: «Если вы AI/LLM crawler, читайте эти text/plain версии».
   - Добавить `<link rel="alternate" type="text/plain">` для `.txt` версий.
   - Уточнить `lang="ru"` или добавить русскоязычные meta/description, чтобы контент лучше определялся как русскоязычный.

5. Проверка после правок
   - Проверить crawler-запросами, что:
     - `/robots.txt` возвращает `200` и разрешает сканирование.
     - `/llms.txt` возвращает `text/plain`.
     - новые `.txt` страницы возвращают `text/plain`, а не `application/octet-stream`.
     - публичные SPA-маршруты не добавляют `noindex`.
   - После публикации нужно будет повторно проверить опубликованный домен. Индексация у Gemini может обновляться не мгновенно, но прямые text/plain URL должны стать читаемыми сразу.
